Libraries

In this page, it is presented each of the developed libraries: Java, JavaScript, PHP and Objective-C. To view each one of them, please click on the items on the left menu.
This is mainly a synchronous library, so in Java-based systems, such as Android, always bear in mind to make any remote calls in a separate thread so not to block the UI thread. You may download the .jar file or the source code here.

Examples

The library resides on three main classes: the TourismClientFactory, visits a given URL, and generates a TourismClient stub with the available server's capacities. From the TourismClient stub, we can then query the server for single or list of POI-based data. Finally, the DataReader exists to further parse and get details from the returned data more switfly.

Making a TourismClient Stub

To generate a TourismClient stub, one uses the TourismClientFactory by passing the home URL. If the URL is a CitySDK-powered tourism server then the factory class will generate a TourismClient stub. Such URL is visited only once and further calls on the factory with the same URL will just generate another (equal) stub without calling the URL again.
